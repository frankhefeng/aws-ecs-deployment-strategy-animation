üéØ Deployment Strategies & Traffic Control

  • Rolling: Task-based replacement, no load balancer required, simplest setup
  • Blue/Green (CodeDeploy): Uses 2 Target Groups, instant switch (0% ‚Üí 100%) by changing ALB listener rules
  • Linear (ECS): Uses 2 Target Groups with weighted routing to shift traffic gradually at fixed intervals (e.g., 10% every minute)
  • Canary (ECS): Uses 2 Target Groups, tests with 10% first, then gradually shifts remaining traffic
  • Key Difference: Blue/Green = instant switch; Linear/Canary = gradual weight adjustment

üí° Key Insights

  • Rolling: Simplest setup, no ALB needed, but slowest rollback
  • Blue/Green: Fastest deployment and instant rollback, but most expensive (2x resources)
  • Linear: Predictable and controlled, good for scheduled deployments
  • Canary: Lowest blast radius (10% users), best for high-risk changes
  • Best Practice: Use Rolling for simple deployments, Canary for unknown risks, Blue/Green when instant rollback is critical, Linear for predictable rollouts
